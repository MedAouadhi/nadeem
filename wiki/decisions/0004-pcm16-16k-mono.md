# 0004 — 16 kHz mono PCM16, 20 ms frames for chat WebSocket

**Status:** accepted
**Date:** 2026-04-25 (recorded)

## Context

Pro mode streams live mic up and TTS audio down. Need an audio format that:
- ESP32-S3 can capture / play without codecs/transcoding.
- Most ASR engines accept natively (Whisper, Azure, Deepgram).
- Keeps WebSocket frames small enough to limit jitter.
- Matches `bsp_init`'s default capture rate (16 kHz).

## Decision

- **Codec:** signed 16-bit little-endian PCM, mono.
- **Sample rate:** 16 kHz.
- **Frame:** 20 ms = 320 samples = 640 bytes per binary WebSocket message upstream.
- **Downstream:** any chunk size; firmware feeds whatever it gets directly to DAC.
- **Control frames:** text JSON, scanned for substrings `"listening"` / `"speaking"`.

## Consequences

- No on-device compression; raw bandwidth ≈ 256 kbps. Acceptable on home Wi-Fi.
- ASR engines work without resampling.
- LEDs driven by simple substring match in control frames — no schema lock-in for backend telemetry.
- No headers / timestamps — just raw samples. Out-of-band reordering not handled (WebSocket TCP guarantees order).
